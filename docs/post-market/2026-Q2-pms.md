# Post-Market Surveillance Plan — Q2 2026

**Product:** MeDoc Dental Practice Management Software
**Class:** MDR Class I (administrative, no diagnostic claim)
**Reporting period:** 2026-04-01 – 2026-06-30
**Plan owner:** Quality Manager (TBD)

## 1. Scope

Continuous monitoring of MeDoc software in production deployments to detect:

- Safety incidents (patient harm, data loss, mis-prescription).
- Security incidents (unauthorised access, data exfiltration).
- Performance regressions (UI > 200 ms p95, command > 500 ms p95).
- Usability complaints likely to lead to use errors.

## 2. Data sources

| Source                        | Frequency         | Owner          |
|-------------------------------|-------------------|----------------|
| Crash telemetry (opt-in)      | Real-time         | DevOps         |
| Customer support tickets      | Daily triage      | Support lead   |
| In-app feedback button        | Weekly review     | Product        |
| `audit_log` anomalies         | Weekly review     | Security       |
| Vulnerability advisories      | On publication    | Security       |
| Dental association feedback   | Quarterly meeting | Quality Mgr    |

## 3. KPIs

- **Incidents reportable to BfArM** — target 0; threshold 1 triggers CAPA.
- **MTTR for security advisories** — high ≤ 7 d, critical ≤ 48 h.
- **Crash-free sessions** — ≥ 99.5 %.
- **Support tickets / 100 active installations** — track trend.

## 4. CAPA process

All findings flow into `docs/post-market/capa-tracking.md`. Each CAPA must
include: trigger, root cause analysis, corrective action, effectiveness
verification, closure approval.

## 5. Vigilance reporting

Reportable serious incidents must be filed with BfArM within 15 days
(MDR Art. 87). Trend reports filed quarterly. PSUR maintained per Art. 86.

## 6. Review

This plan is reviewed at the end of every quarter. The next review is due
**2026-07-15**. Output: amended plan + summary report archived under
`docs/post-market/2026-Q2-pms-report.md`.
